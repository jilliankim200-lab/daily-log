# 최적가이드 페이지 — 요구사항 & 테스트 시나리오

## 1. 요구사항 체크리스트

### 1-1. 기본 플랜 로직

- [x] 계좌별 액션플랜 표시 (매도 → 재매수 스텝 구분)
- [x] 동일 소유자 내 중복 종목 감지 → 절세 우선순위 낮은 계좌에서 매도 권고
  - 우선순위: IRP > 퇴직연금 > 연금저축 > ISA > 일반
  - 예외: 수익률 40% 이상 종목은 매도 대상 제외 (★유지)
- [x] 매도 후 가용현금 = 매도금액 + 기존현금
- [x] 재매수: 목표 비중(주식/채권/커버드콜/금/기타) 기반으로 keep 종목에 비례 배분
- [x] 가용현금 없는 계좌는 "변경 없음" 표시
- [x] 목표 비중 KV(`rebalancing_targets`)에서 로드, localStorage fallback
- [x] 이전 목표 비중(`rebalancing_targets_prev`) 대비 커버드콜 배분 변화 표시

### 1-2. 퇴직연금 안전자산 비중 체크

- [x] 퇴직연금(IRP/DC/DB) 계좌 자동 감지
- [x] 안전자산(채권+금) 비중 30% 미만 → `under` 경고 표시
- [x] 안전자산 비중 35% 초과 → `over` 경고 표시
- [x] 30~35% → `good` 상태
- [x] 비중 프로그레스 바 (30% / 35% 마커 표시)
- [x] 부족액 / 초과액 → 구체적 금액으로 안내

### 1-3. 매수 타이밍 신호 (TimingBadge)

- [x] 종목별 주가 신호 fetch (fetchStockSignals)
- [x] 매도 종목: 매도 타이밍 배지 표시
- [x] 재매수 종목: 매수 타이밍 배지 표시
- [x] TimingBadge: 레이블만 기본 표시, hover 시 툴팁에 가격 범위 + 설명 표시
  - 가격 범위: `15,400원 ~ 19,000원` 형식 (만 단위 축약 사용 안 함)
- [x] 신호 필터: 특정 배지(분할매수/조정대기 등) 클릭 → 해당 신호만 강조, 나머지 dim
- [x] 신호 범위 기준 모달 (? 버튼) — 매도/매수 배지별 기준 설명

### 1-4. 커버드콜 월 배당금 추정

- [x] 현재 CC 보유 기준 월 배당 수익률 계산
- [x] 목표 비중의 CC 금액 기준으로 목표 배당금 추정
- [x] USD 배당은 세후(×0.85) × 환율 적용

### 1-5. 행(Row) UI 레이아웃

- [x] 배지(유지/추가매수/전량매도) — 종목명 + 자산분류 — 평가수익률·평가금액 — 타이밍배지
- [x] 서브라인: 추가매수 `+N만 투자 → 약 N주 매수 가능`
- [x] 수익률/금액 레이블 명시: `평가수익률 +1.8% · 평가금액 642만`
- [x] 모든 텍스트 13px 이상 (9px 이하 금지)

### 1-6. 추천배당주 모드 (divRecMode)

- [x] 헤더에 "추천배당주" 토글 버튼
  - 활성 색상: `var(--accent-blue-fg)` (파란 텍스트)
  - 비활성 색상: `var(--asset-covered)` (CC 색)
- [x] 첫 활성화 시 KV `dividend_analysis` 로드
- [x] 추천 종목 중 prices에 없는 티커 → `/stock-prices` Worker 엔드포인트로 추가 fetch
- [x] 문제 CC 종목 (riskLevel: high/medium) 식별
  - high → 30% 축소 권고
  - medium → 15% 축소 권고
- [x] 문제 CC 서브라인: `↓N주 축소 권고 (M만원)` (주황 경고색)
- [x] 축소 재원으로 추천 종목 신규 매수 배분
  - 추천 수로 균등 분배
  - `+N만 투자 → 약 N주 매수 가능` 형식
- [x] **재매수 계좌**: keeps 아래 "배당분석 추천 신규 매수" 섹션
- [x] **변경없음 계좌**: 변경없음 keep 목록 아래에도 추천 섹션 표시
- [x] 추천 종목 행: `N위` 배지 + 종목명 + 추천 이유 + 매수 수량·금액 + 월배당률 + 방어율

## 2. 테스트 시나리오

### TC-01. 중복 종목 매도 플랜

**전제**: KODEX 미국채10년이 일반계좌(ISA)와 IRP 두 곳에 보유

- [ ] ISA 계좌에 `전량매도` 배지로 표시되는가
- [ ] IRP 계좌에는 `유지` 배지로 표시되는가
- [ ] STEP 1 헤더에 매도 총금액이 표시되는가
- [ ] STEP 2 헤더에 `가용현금 N원 (매도 N + 기존현금 N)` 표시되는가

### TC-02. 수익률 40% 이상 예외 처리

**전제**: 중복 종목인데 수익률이 45%

- [ ] 해당 종목이 매도 대상에서 제외되는가
- [ ] `★유지` 배지(금색)로 표시되는가

### TC-03. 퇴직연금 안전자산 비중 경고

**전제**: IRP 계좌의 안전자산(채권+금) 비중이 20%

- [ ] 카드 배경이 경고색(주황)으로 강조되는가
- [ ] `안전자산 비중 XX% / 목표 30~35%` 표시되는가
- [ ] `채권/금 추가 매수 필요: NNN만` 안내가 나오는가
- [ ] 프로그레스 바에 30%, 35% 마커가 보이는가

### TC-04. 매수 타이밍 배지 & 가격 범위

**전제**: 재매수 종목에 시세 신호 데이터 존재

- [ ] 각 종목 오른쪽에 타이밍 배지(분할 매수/조정 대기 등)가 표시되는가
- [ ] 배지 hover 시 툴팁이 나타나는가
- [ ] 툴팁 상단에 `15,400원 ~ 19,000원` 형태의 가격 범위가 보이는가
- [ ] 툴팁 하단에 텍스트 설명이 보이는가
- [ ] 배지 자체에는 가격 범위 숫자가 없는가 (레이블만 표시)

### TC-05. 신호 필터

**전제**: 여러 종목에 다양한 배지 존재

- [ ] 상단 필터 버튼에 현재 포트폴리오에 존재하는 신호만 표시되는가
- [ ] `분할 매수` 필터 클릭 시 해당 종목만 강조, 나머지 opacity 낮아지는가
- [ ] 동일 버튼 재클릭 시 필터 해제되는가
- [ ] 매도 필터 활성화 시 재매수 섹션이 전체 dim 처리되는가

### TC-06. Row 레이아웃 가독성

- [ ] 배지 — 종목명 — 오른쪽 수치 순서가 맞는가
- [ ] 오른쪽 수치: `평가수익률 +X% · 평가금액 NNN만` 한 줄인가
- [ ] 서브라인: `+NNN만 투자 → 약 N주 매수 가능` 이 종목명 아래 회색으로 나오는가
- [ ] 모든 텍스트가 13px 이상인가 (개발자도구 확인)

### TC-07. 추천배당주 버튼 토글

**전제**: dividend_analysis KV에 데이터 존재

- [ ] 버튼 클릭 전: `var(--asset-covered)` 색상으로 표시되는가
- [ ] 버튼 클릭 후: `var(--accent-blue-fg)` 색상으로 변경되는가
- [ ] 최초 클릭 시 KV 로딩이 발생하는가 (Network 탭 확인)
- [ ] 재클릭 시 KV를 다시 호출하지 않는가 (캐시 확인)
- [ ] 추천 종목 중 기존 prices에 없는 티커가 있으면 `/stock-prices?tickers=...` 호출되는가

### TC-08. 추천배당주 — 재매수 계좌

**전제**: 문제 CC(high/medium) 종목이 재매수 섹션에 있는 계좌

- [ ] 해당 CC 종목의 서브라인에 `↓N주 축소 권고 (M만원)` 표시되는가
- [ ] high는 30%, medium은 15% 비율로 계산되는가
- [ ] keeps 목록 아래 구분선 후 "배당분석 추천 신규 매수" 섹션이 나오는가
- [ ] 추천 종목마다 `N위` 배지, 종목명, 추천 이유, 매수 수량, 월배당률, 방어율이 보이는가
- [ ] 매수 수량: (전체 축소 재원 ÷ 추천 수) ÷ 현재가 로 계산되는가

### TC-09. 추천배당주 — 변경없음 계좌

**전제**: 매도 없는 계좌에 문제 CC 종목 보유

- [ ] 변경없음 keep 목록에서도 문제 CC 종목에 `↓N주 축소 권고` 서브라인이 나오는가
- [ ] keep 목록 아래에 "배당분석 추천 신규 매수" 섹션이 나오는가 (재매수 계좌와 동일하게)
- [ ] 추천 섹션이 없는 계좌(문제 CC 없음)에는 해당 섹션이 표시되지 않는가

### TC-10. 추천배당주 — 가격 없는 추천 종목

**전제**: 추천 종목의 현재가를 가져오지 못한 경우

- [ ] 주수 대신 `N만 투자` 형식으로 표시되는가
- [ ] 에러 없이 렌더링되는가

### TC-11. 다크모드

- [ ] 행 배경에 빨간/녹색 배경색이 없는가 (수익률은 텍스트 색상으로만)
- [ ] 모든 배지가 다크 배경에서 가독성이 있는가

## 3. 고려된 설계 사항

| 항목 | 내용 |
|------|------|
| 중복 기준 | 소유자(owner) 단위, 계좌 단위 아님 |
| 절세 우선순위 | IRP(5) > 퇴직연금(4) > 연금저축(3) > ISA(2) > 일반(1) |
| 비중 배분 방식 | 자산분류별 목표비중 → 해당 분류 내에서 현재 평가금액 비례 |
| 안전자산 정의 | 채권 + 금 (현금 제외) |
| 퇴직연금 판정 | accountType 또는 alias에 퇴직/IRP/DC/DB 포함 여부 |
| 수익률 계산 | (현재가 - 평균단가) / 평균단가 × 100, 가격 없으면 null |
| 펀드 처리 | isFund=true → h.amount 직접 사용, 주수 계산 없음 |
| 가격 소스 | prices context (Worker daily-prices) → 없으면 avgPrice fallback |
| CC 축소 재원 | 계좌 내 문제 CC 총 축소 금액 → 추천 수로 균등 분배 |
| 추천 가격 소스 | prices context → recPrices(별도 fetch) 순서로 fallback |
