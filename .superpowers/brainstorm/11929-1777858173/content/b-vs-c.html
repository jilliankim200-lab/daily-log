B vs C — 실제 파일 구조로 비교

어떤 구조로 코드가 달라지는지 직접 보여드립니다

B — fetchStockSignals 확장
📁 src/utils/
fetchStockSignals.ts  수정
getSellSignal()    ← 기존 (타이밍)
getSellDecision() ← 신규 추가
SELL_CONFIG = {
  targetReturn: 0.20,
  stopLoss: -0.10
}
장점: 파일 1개만 건드림
단점: 데이터 유틸인데 판단 로직까지 담당
        → 나중에 설정 바꾸려면 이 파일 찾아야 함
C — Config + Engine 분리
📁 src/utils/
fetchStockSignals.ts  그대로
sellConfig.ts         신규
targetReturn: 0.20
stopLoss: -0.10
riskTable: {...}
sellEngine.ts        신규
getSellDecision(holding, prices)
→ { action, reason, urgency }
장점: 설정·로직 분리 → 각자 역할 명확
        목표% 바꿀 때 sellConfig만 열면 됨
단점: 파일 2개 (but 각 파일은 짧고 명확)
제 추천은 C입니다.

fetchStockSignals.ts 어딘가를 뒤져야 합니다.
